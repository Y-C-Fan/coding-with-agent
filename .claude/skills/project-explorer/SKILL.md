## 项目探索流程

当需要探索和理解一个项目时，请按以下步骤进行：

### 1. 目录结构分析
- 列出顶层目录和文件
- 识别项目类型（前端、后端、库、工具等）
- 了解目录划分逻辑

### 2. 核心文件识别
- package.json / Cargo.toml / requirements.txt 等依赖文件
- 配置文件（.gitignore, .env.example, config.* 等）
- 构建脚本（Makefile, CMakeLists.txt, build.* 等）
- 入口文件（main.*, index.*, app.* 等）

### 3. 技术栈识别
- 编程语言和版本
- 框架和库
- 数据库、缓存等技术
- 构建工具

### 4. 依赖关系分析
- 主要模块的依赖关系
- 核心功能的位置
- 配置和入口点

### 5. 运行方式
- 如何安装依赖
- 如何运行项目
- 是否有测试，如何运行

## 输出格式

请生成以下格式的项目分析报告：

```
# 项目分析报告

## 概述
[项目类型、功能简介]

## 技术栈
- 语言/框架: 
- 依赖管理: 
- 其他技术: 

## 目录结构
[关键目录及其作用]

## 核心模块
[主要模块及功能]

## 运行方式
[安装和运行指令]

## 注意事项
[特殊配置、环境要求等]
```
